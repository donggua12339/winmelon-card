# 安全政策

## 报告漏洞

我们非常重视 WM 官方虚拟卡密交易平台的安全。如果你发现漏洞，请**私下披露**：

- 邮件：security@your-domain.example
- **请勿公开提交 GitHub Issue**
- 请包含：复现步骤、影响范围、建议修复方案（如有）

## 响应时间

- 收到报告后 48 小时内确认。
- 7 天内评估影响。
- 关键漏洞 7 天内修复，14 天内发布修复版本。
- 修复发布 7 天后公开披露。

## 适用范围

接收任何以下方面的报告：
- 服务端漏洞（SQL 注入、RCE、越权、认证绕过等）
- 客户端漏洞（XSS、CSRF 等）
- 业务逻辑漏洞（支付绕过、卡密白嫖、并发超卖等）
- 供应链漏洞（依赖项）

**不在范围**：
- 自报家门式的弱点扫描报告。
- 已知漏洞的批量重复提交。
- 社会工程攻击。
