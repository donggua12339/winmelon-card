# 贡献指南

感谢你为 WM 官方虚拟卡密交易平台贡献！

## 开发流程

1. Fork 仓库，从 `develop` 创建功能分支 `feature/<name>`。
2. 遵循 [CODING-STANDARD.md](./CODING-STANDARD.md) 与 [SECURITY-BASELINE.md](./SECURITY-BASELINE.md)。
3. 提交规范见 [CODING-STANDARD.md §8](./CODING-STANDARD.md#8-git-提交规范)。
4. PR 必须 ≤ 500 行 diff，CI 全绿，关联 issue。
5. 安全相关 PR 加 `security` 标签，紧急评审。

## 本地启动

见 [README.md](./README.md#快速开始)。

## 安全漏洞报告

**请勿在 PR / Issue 中公开漏洞细节**。详见 [SECURITY.md](./SECURITY.md)。

## 行为准则

请保持友善、专业、互相尊重。
